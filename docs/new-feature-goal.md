# 🎯 Feature Specification: "The Goal" (꿈을 담는 그릇)

## 1. 기능 소개 (Introduction)
'The Goal'은 사용자가 저축 목표(예: 여행, 전자기기 구매, 비상금 마련 등)를 설정하고, 현재 잔액이 그 목표에 얼마나 다다랐는지 실시간으로 시각화해 주는 기능입니다. 단순한 숫자의 나열을 넘어, **자산의 증가가 곧 꿈의 실현**으로 이어짐을 직관적으로 느끼게 합니다.

## 2. 상세 UX 시나리오 (The Journey)

### 2.1 진입 및 목표 설정 (Setting the Vision)
* **트리거:** 메인 화면의 [인사이트 그래프] 하단에 "목표를 설정하고 동기부여를 받아보세요 [+]" 라는 미니멀한 버튼이 노출됩니다. (최소 1개 이상의 데이터가 있을 때만 노출)
* **입력 시트:** 버튼 클릭 시 아래에서 위로 '목표 설정 시트'가 올라옵니다.
    * **목표 명:** 사용자 자유 입력 (예: 💻 맥북 프로 사기)
    * **목표 금액:** 숫자 입력 (시스템 키보드)
    * **이모지:** 목표를 상징하는 아이콘 하나를 선택.
* **완료:** 저장 시 시트가 닫히며, 메인 대시보드에 'Liquid Progress Bar'가 생성됩니다.

### 2.2 실시간 인터렉션 (The Living Flow)
* **잔액과의 동기화:** 사용자가 수입(Income)을 추가하면 게이지가 오른쪽으로 부드럽게 차오르고, 지출(Expense)이 발생하면 왼쪽으로 줄어듭니다.
* **액체 애니메이션:** 게이지가 찰렁이는 듯한(Liquid) 효과를 주어 데이터가 살아있는 유기체처럼 느껴지게 합니다.
* **정보 노출:** 바 위에는 "목표까지 **1,240,000원** 남았어요" 혹은 "**70%** 달성"과 같은 문구가 연한 회색으로 표시됩니다.

### 2.3 달성 및 보상 (The Celebration)
* **100% 도달:** 현재 잔액이 목표 금액과 같거나 커지는 순간, 게이지가 황금빛으로 반짝입니다.
* **시각적 보상:** 화면 전체에 아주 미세한 **골드 & 화이트 컨페티(Confetti)** 애니메이션이 1.5초간 흩날리며 사용자에게 성취감을 선사합니다.
* **아카이빙:** 달성된 목표는 축하 메시지와 함께 '완료됨' 상태로 변경되며, 사용자는 새로운 목표를 설정할 수 있습니다.

## 3. DDR (Design Decision Records): 설계 배경 및 결정 사항

| ID | 결정 사항 (Decision) | 결정 배경 및 이유 (Rationale) |
| :--- | :--- | :--- |
| **DDR-G01** | **Liquid Progress Bar 도입** | 단순한 막대보다 액체가 차오르는 듯한 시각 효과가 자산의 '축적'이라는 개념을 더 풍부하게 전달함. |
| **DDR-G02** | **이모지(Emoji) 활용** | 텍스트만 있는 목표보다 시각적 심볼(💻, ✈️, 🏠)이 포함될 때 사용자의 감정적 몰입도가 훨씬 높음. |
| **DDR-G03** | **컨페티(Confetti) 피드백** | 금융 앱은 자칫 딱딱해질 수 있음. 달성 순간의 확실한 시각적 보상은 사용자가 앱을 계속 쓰고 싶게 만드는 '도파민 장치'가 됨. |
| **DDR-G04** | **비례적 잔액 연동** | 목표 전용 계좌가 따로 있는 것이 아니므로, 전체 잔액(Total Balance)을 기준으로 목표 달성률을 계산하여 '현재 내가 가진 돈의 가치'를 재정의함. |

## 4. 개발자를 위한 기술 명세 (Implementation Specs)

### 4.1 데이터 스키마 추가
```typescript
interface Goal {
  id: string;
  title: string;       // 목표 명칭
  targetAmount: number; // 목표 금액
  emoji: string;       // 선택한 이모지
  isActive: boolean;    // 현재 활성화 여부
  achievedAt?: string;  // 달성 시각 (타임스탬프)
}
```

### 4.2 핵심 로직
* **진행률 계산:** `(현재 잔액 / 목표 금액) * 100` (단, 100%를 초과할 수 있음)
* **조건부 렌더링:** `Goal` 객체가 존재하고 `isActive`가 true일 때만 메인 대시보드 하단에 컴포넌트 노출.
* **애니메이션 라이브러리:** Framer Motion 혹은 CSS Keyframes를 활용하여 게이지의 찰렁임과 컨페티 효과 구현.
